KeFVP paper (baseline)

proposes a two-phase framework for financial volatility prediction.

Phase 1 – Language Model Pretraining:
For each financial metric (FM), the authors collect its definition from Wikipedia and identify occurrences of that metric within financial texts. They then concatenate the corresponding definition to each sentence containing the FM and pretrain a BERT model using the MLM objective. This approach is intended to help the model better understand financial metrics by explicitly incorporating their definitions. However, similar approaches (e.g., FinBERT) have already demonstrated this idea.

Phase 2 – Financial Volatility Prediction (FVP):
In this phase, the authors compute historical volatilities and obtain text embeddings for lines in ECCs. They adopt the Autoformer architecture.

InstructSpeech (paper)

InstructSpeech serves as a proof of concept demonstrating that separating the acoustic and semantic feature modules can improve downstream speech-based tasks. In this paper, the researchers design an architecture that explicitly separates the analysis of acoustic and semantic features for performing the speech editing task. The semantic tokens are obtained by extracting embeddings from the 12th layer of HuBERT and then applying k-means clustering to discretize them. Since the lower layers of HuBERT primarily capture acoustic properties while the higher layers capture semantic information, using the 12th-layer embeddings effectively represents the speech’s semantic content. For the acoustic features, the model uses tokens produced by the SoundStream tokenizer, allowing it to separately handle aspects such as timbre, emotion, and prosody.

  • Task3

  R&D on embeddings of Qwen2-audio (technical report) with QAs

Qwen2-Audio is a LALM composed of three main components: the audio tower, the multi-modal projector, and the LM.
Audio tower: The model does not take the raw waveform directly into the audio tower; instead, the waveform first passes through a Whisper encoder that converts it into log-mel spectrogram features. These features are then processed by the audio tower, a transformer-based encoder that captures high-level acoustic and phonetic representations from the speech signal.
Multi-modal projector: The output of the audio tower is then passed to the multi-modal projector, implemented as a linear transformation that maps the 1280-dimensional audio features to the 4096-dimensional text embedding space used by the LM. This layer performs the essential alignment between audio and text modalities, allowing the LM to interpret projected audio embeddings in the same representational space as text tokens.
LM: Finally, the LM block takes both the text tokens and the projected audio embeddings as input and fuses them into a unified contextual representation that reflects information from both modalities.

In my setup, I use the vector output from the LM block, which encodes intertwined information from both the input audio and text. For each QA pair, the audio input corresponds to the spoken segment from the transcript that contains the answer, while the text prompt is designed to guide the model toward an embedding space that emphasizes acoustic rather than semantic characteristics. Task 6


We want to generate a Qwen embedding that represents the specific question–answer pair. The goal is to extract features that capture:

  • a: What information is being discussed (content)
  • b: How the information is expressed (emotion / tone / sentiment)

To do this task, we should include the QA in the prompt:
Qwen_embedding = embedding(Answer audio| given QA, ECC)

embedding(audio | given context, given QA)

We used Qwen-2.5-Omni-7B with 4-bit NF4 quantization, float16 computation, and double quantization for efficient inference. The model architecture consists of a Thinker module and a Talker module. For feature extraction, we used the last hidden state of the Thinker module, which has the shape (batch size, sequence length, 3584), where 3584 is the hidden feature dimension. We chose the Thinker output because it represents the model's internal contextualized understanding of the multimodal input (audio and text) before any generative decoding is performed. This hidden representation is therefore better suited for embedding extraction than the Talker output, which is optimized for generating responses. To obtain a fixed-length embedding, we applied mean pooling across the sequence dimension. However, directly averaging all token representations would allow PAD tokens to influence the final embedding. Since the lengths of the input transcripts and audio segments vary across samples, the number of PAD tokens also varies, introducing inconsistent noise into the pooled representation. To address this, we used masked mean pooling, averaging only the hidden states corresponding to non-padding tokens as indicated by the attention mask.

To guide the model toward capturing both the semantic context and the speaker's delivery, we designed a prompt that combines the audio segment with its associated topic, question, and answer transcript. The system prompt instructs the model to focus primarily on acoustic and emotional characteristics—such as tone, emotion, pace, confidence, hesitation, and volatility—rather than the literal content of the speech. By conditioning the audio analysis on the question and transcript context, the resulting embedding is intended to capture both what is being discussed and how it is being expressed, producing a richer representation of each Earnings Conference Call response.

Ablation

To evaluate the contribution of audio information, we perform an ablation study in which embeddings are generated under three different settings.

First, we generate embeddings using only the transcript text, without providing audio to Qwen. This serves as the text-only baseline.

Next, we generate multimodal embeddings using both transcript and audio. To investigate whether prompt design affects the usefulness of audio information, we evaluate two different prompts.

Prompt 1 is a general acoustic-emotional analysis prompt. It instructs the model to focus on how the speaker delivers the message, including characteristics such as tone, emotion, pace, confidence, and hesitation, rather than the literal content of the speech.

Prompt 2 is designed to be more closely aligned with the downstream task of stock volatility prediction. It provides the topic and financial question associated with the earnings call segment and explicitly instructs the model to analyze both the content and delivery of the segment with a focus on characteristics that may be relevant to future stock volatility.

The resulting embeddings from these three settings (text-only, audio + Prompt 1, and audio + Prompt 2) are then used within the same volatility prediction framework. This allows us to isolate the effect of incorporating audio information and assess whether task-specific prompting improves the usefulness of multimodal embeddings for volatility forecasting. All variants use the same downstream forecasting framework, which combines ECC embeddings with a set of multi-horizon historical volatility features derived from realized volatility measures. The only difference between the three variants is the embedding generation strategy. Since the downstream model and volatility features remain unchanged across all settings, performance differences can be attributed solely to the information captured by the ECC embeddings.

The ablation study demonstrates that audio information provides complementary signals beyond textual content when incorporated through a task-aware prompting strategy. On both MAEC-15 and MAEC-16, the volatility-oriented multimodal prompt (Prompt 2) achieves the lowest average MSE, reducing prediction error from 0.196 to 0.188 on MAEC-15 and from 0.295 to 0.288 on MAEC-16. In contrast, a generic emotion-oriented prompt (Prompt 1) improves performance on MAEC-15 but degrades performance on MAEC-16. These findings suggest that the value of audio information depends strongly on how the multimodal model is guided during representation extraction, with task-specific prompting enabling the model to capture volatility-relevant acoustic and semantic cues more effectively.
