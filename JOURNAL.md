# JOURNAL.md — Work Journal

**Append-only, newest entry last.** A chronological record of work performed on the project: what was done, what was found, what broke, what's next. The journal is the project's narrative memory — [TASKS.md](TASKS.md) says *what state* the work is in; this file says *how it got there*.

## What gets an entry

Append an entry at the end of every working session (or completed task, whichever is finer-grained). Include:

- **Date + scope** — what task(s)/topic the session touched (use task IDs where applicable).
- **Done** — work completed, with enough specifics that a future reader can retrace it (commands, files, run IDs).
- **Found** — results, measurements, surprises, dead ends. Negative findings are first-class: an experiment that failed is information.
- **Sources** — papers/links/model cards consulted (anything reusable should also land in DESIGN.md §13).
- **Next** — the intended next step, so any session can pick up where the last one stopped.

Keep entries factual and compact. Decisions that change the design belong in [DECISIONS.md](DECISIONS.md) (link the journal entry to them); the journal records the work and evidence around them.

### 2026-06-12 — Exploration: FinCall-Surprise identity reconstruction feasibility
- **Done:** Discovered while scoping T1.2 that FinCall-Surprise records carry **no ticker, company name, or call date** (opaque numeric IDs; Seeking Alpha headers stripped from `input`). Probed recovery routes: SA article-ID URLs (404), EarningsCast — the paper's audio source — is dead entirely (HTTP 410 site-wide), mp3 ID3 tags (empty), slide PDFs (rich: `/Company` metadata, title-page text, dates). Per user decision (DECISIONS.md), ran a 50-call seeded feasibility study (`notebooks/explore_fincall_identity.py`, seed 0): extract name candidates from PDF metadata/page-1 + operator-greeting regex, match against SEC `company_tickers.json` (8,007 names), extract dates from PDF page-1 text + creation stamps.
- **Found:** (1) **Date recovery: 50/50 (100%)** — PDF page-1 date or creation stamp present on every sampled call (PDFs cover 2,671/2,688 calls). (2) **Ticker via quick heuristics: 29/50 (58%)**, but the bottleneck is name→ticker *matching*, not signal: manual read of all misses confirms the company is named plainly in the transcript's first ~700 chars in 10/10 no-candidate cases (Arista, International Paper, DXC, DowDuPont, Vulcan, TransDigm, Stanley Black & Decker…). Failures are brand-vs-legal-name gaps (ADP/Citi/ITW/Otis/PPG vs "Automatic Data Processing"/"Citigroup"/"Illinois Tool Works"/…) — fixable with an alias table and/or LLM-assisted name extraction + deterministic matching. Realistic ceiling ≥95%. (3) **New risk: corpus heterogeneity** — the sample + earlier spot-reads contain non-earnings calls (JPMorgan tech-forum fireside with Qualcomm, DXC/Luxoft M&A announcement, Ford monthly-sales call, BMS/Celgene merger call). The "2,688 conference calls" are not all earnings calls; T1.4 needs a call-type classifier + exclusion reason codes, and the per-call `label` (earnings surprise) may be the cleanest earnings-call indicator to cross-check. (4) SEC `company_tickers.json` (free, official, name+ticker+CIK) works well as the matching backbone.
- **Sources:** arXiv 2510.03965 (FinCall-Surprise paper: transcripts from Seeking Alpha, audio EarningsCast, slides Bloomberg/company sites; identity table kept internal for IBES join); sec.gov/files/company_tickers.json.
